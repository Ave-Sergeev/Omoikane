## Omoikane

[Русский](https://github.com/Ave-Sergeev/Omoikane/blob/main/README.md) | [English](https://github.com/Ave-Sergeev/Omoikane/blob/main/README.en.md)

### Описание

Данный проект представляет собой легковесный `Explicit Proxy` написанный на `Rust`.  
Благодаря архитектурным особенностям языка обеспечиваются минимальные задержки и низкое потребление ресурсов системы.

**Основная задача**:  
Сохранение устойчивости TCP-соединений в условиях глубокого анализа трафика (DPI) на промежуточных узлах сети, а также применение методов защиты от атак типа DNS Spoofing и Cache Poisoning.

**Дисклеймер**:  
Настоящее программное обеспечение разработано в рамках подготовки магистерской диссертации и носит исключительно научно-исследовательский характер.
Разработка представлена как `Proof of Concept` (доказательство концепции) с целью исследования механизмов обеспечения устойчивости информационного обмена при прохождении через узлы с глубоким анализом трафика (DPI).

Программное обеспечение предоставляется на условиях `as is` (как есть). Его использование допускается только в учебных и ознакомительных целях.
Автор не дает никаких гарантий относительно работоспособности инструмента в специфических условиях и не несет ответственности за любой прямой или косвенный ущерб, возникший в результате использования данного ПО.

**Текущее состояние**:  
Active Research & PoC 🦀  
На данном этапе проект скорее является Proof of Concept, чем готовым решением.  
Логика манипуляции TCP-трафиком, архитектура и общий функционал требует доработки и оптимизации.  

Целевая платформа: macOS (aarch64-apple-darwin) — Apple Silicon (процессоры M1/M2/M3/...).  
Статус: Тестирование и стабильная работа подтверждены только на macOS автора.

В данном репозитории не соблюдаются формальные промышленные стандарты ведения Git-истории (Best Practices).

Проект назван в честь японского мифологического бога разума, мудрости и стратегии, который вернул миру свет, найдя «хитрый способ» там, где прямое действие было бессильно.

### Конфигурация CLI

Настройка сервиса осуществляется через аргументы командной строки.  
Список доступных опций:  

- `APP`
  - `--addr` - ip-адрес для прослушивания. (По умолчанию: `127.0.0.1`)
  - `--port` - порт для прослушивания. (По умолчанию: `8080`)
  - `--log-level` - уровень детализации логов: `off`, `error`, `warn`, `info`, `debug`, `trace`. (По умолчанию: `info`)
- `DNS`
  - `--dns-mode` - режим работы DNS: `system`, `doh`, `dot`. (По умолчанию: `system`)
  - `--dns-qtype` - тип запрашиваемых DNS записей: `ipv4`, `ipv6`, `all`. (По умолчанию: `ipv4`)
  - `--dns-provider` - используемый провайдер для DoH/DoT: `google`, `cloudflare`, `quad9`. (По умолчанию: `google`)
- `HTTP`
  - `--http-split-mode` - фрагментация HTTP-request: `none`, `fragment`. (По умолчанию: `none`)
- `HTTPS`
  - `--https-fake-ttl` - стратегия работы с TTL для фейк пакетов: `none`, `auto`. (По умолчанию: `none`)
  - `--https-split-mode` - фрагментация TLS ClientHello: `none`, `fragment`. (По умолчанию: `none`)

### Быстрый старт

Самый быстрый способ начать работу — скачать скомпилированный бинарный файл для вашей системы:  
1. Перейдите на страницу [Releases](https://github.com/Ave-Sergeev/Omoikane/releases).
2. Скачайте архив для вашей ОС.
3. Распакуйте архив и переместите бинарный файл куда вам удобно. Запустите его из терминала.

### Сборка

Для самостоятельной сборки необходимо наличие установленного инструментария `Rust` (компилятор `rustc` и пакетный менеджер `cargo`).  

1. Установите (если необходимо) инструментарий `Rust`.  
    Для Unix-подобных систем (macOS, Linux, ...) выполните команду в терминале:
    ```shell
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```
2. Выполните проверку правильности установки (в терминале).  
    В случае успешной установки (п. 1), вы увидите примерно это `cargo 1.94.0 ...`.
    ```shell
    cargo --version
    ```
3. Клонируйте проект из `GitHub` и перейдите в корень проекта.  
    Выполните сборку в режиме `--release`. После сборки исполняемый файл будет в директории `./target/release/` проекта.  
    ```shell
    cargo build --release
    ```
4. Запустите проект через `cargo` (находясь в корне проекта).  

    ```shell
    cargo run --release -- [аргументы]
    ```
    Или перенести куда вам удобно скомпилированный бинарный файл, и запустите его из терминала.  

    ```shell
    ~/<path_to_binary_file> [аргументы]
    ```

### Примеры запуска в CLI

Все параметры имеют значения по умолчанию. Если не указывать аргументы, будут использованы стандартные настройки.  
- **Базовый режим**. Трафик пропускается без модификаций, используется системный DNS-resolver.  
  Минималистичный запуск (используются дефолтные параметры):  
    > ./<path_to_binary_file>
  
  Полная запись (явное указание параметров):  
    > ./<path_to_binary_file> -a "127.0.0.1" -p "8080" --dns-mode "system" --log-level "info" --http-split-mode "none" --https-split-mode "none" --https-fake-ttl "none"

- **Умеренный режим**. Включает фрагментацию пакетов, Google DoT (IPv4) для обхода простых ограничений.
  > ./<path_to_binary_file> -a "127.0.0.1" -p "8080" --dns-mode "dot" --dns-qtype "ipv4" --https-split-mode "none" --https-split-mode "fragment"

- **Максимальный режим**. Использование DNS-over-HTTPS, фрагментации TLS-пакетов и автоматической подстройки TTL.
  > ./<path_to_binary_file> -a "127.0.0.1" -p "8080" --dns-mode "doh" --dns-provider "cloudflare" --dns-qtype "all" --https-split-mode "fragment" --https-split-mode "fragment" --https-fake-ttl "auto"
