## Omoikane

---

[Русский] | [English](https://github.com/Ave-Sergeev/Omoikane/blob/main/README.en.md)

### Описание

Данный проект представляет собой легковесный `Explicit Proxy` написанный на `Rust`.  
Благодаря архитектурным особенностям языка обеспечиваются минимальные задержки и низкое потребление ресурсов системы.

**Основная задача**:  
Сохранение устойчивости TCP-соединений в условиях глубокого анализа трафика (DPI) на промежуточных узлах сети, а также применение методов защиты от атак типа DNS Spoofing и Cache Poisoning.

**Дисклеймер**:  
Настоящее программное обеспечение разработано в рамках подготовки магистерской диссертации и носит исключительно научно-исследовательский характер.
Разработка представлена как `Proof of Concept` (доказательство концепции) с целью исследования механизмов обеспечения устойчивости информационного обмена при прохождении через узлы с глубоким анализом трафика (DPI).

Программное обеспечение предоставляется на условиях `as is` (как есть). Его использование допускается только в учебных и ознакомительных целях.
Автор не дает никаких гарантий относительно работоспособности инструмента в специфических условиях и не несет ответственности за любой прямой или косвенный ущерб, возникший в результате использования данного ПО.

**Текущее состояние**:  
Active Research & PoC 🦀  
На данном этапе проект скорее является Proof of Concept, чем готовым решением. Логика манипуляции TCP-трафиком, архитектура и общий функционал требует доработки и оптимизации.

**Символизм**:  
Проект назван в честь японского мифологического бога разума, мудрости и стратегии, который вернул миру свет, найдя «хитрый способ» там, где прямое действие было бессильно.

### Конфигурация CLI

Настройка сервиса осуществляется через аргументы командной строки.  
Список доступных опций:  

- `APP`
  - `--addr` - ip-адрес для прослушивания. (По умолчанию: 127.0.0.1)
  - `--port` - порт для прослушивания. (По умолчанию: 8080)
  - `--log-level` - уровень детализации логов: `off`, `error`, `warn`, `info`, `debug`, `trace`. (По умолчанию: `info`)
- `DNS`
  - `--dns-mode` - режим работы DNS: `system`, `doh`, `dot`. (По умолчанию: `system`)
  - `--dns-qtype` - тип запрашиваемых DNS записей: `ipv4`, `ipv6`, `all`. (По умолчанию: `ipv4`)
  - `--dns-provider` - используемый провайдер для DoH/DoT: `google`, `cloudflare`, `quad9`. (По умолчанию: `google`)
- `HTTPS`
  - `--https-fake-ttl` - стратегия работы с TTL для фейк пакетов: `none`, `auto`. (По умолчанию: `none`)
  - `--https-split-mode` - способ фрагментации TLS ClientHello: `none`, `fragment`. (По умолчанию: `none`)

### Сборка проекта

Для сборки необходимо наличие установленного инструментария `Rust` (компилятор `rustc` и пакетный менеджер `cargo`).  

1. Установка на Unix-подобные системы (macOS, Linux, ...).  
Запустите следующую команду в терминале:  

```shell
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```
По окончании загрузки вы получите последнюю стабильную версию инструментария `Rust` для вашей платформы.  

2. Выполнение проверки правильности установки.  
Запустите следующую команду в терминале:  

```shell
cargo --version
```
В случае успешной установки (п. 1), вы увидите примерно это `cargo 1.94.0 ...`.  

3. Клонирование проекта из `GitHub`. Клонируем и переходим в корень проекта.  
Для получения оптимизированного бинарного файла выполняем сборку в режиме `release`.

```shell
cargo build --release
```
После сборки исполняемый файл будет находиться в директории `./target/release/` проекта.  

4. Запуск.  

Вы можете запустить проект через `cargo` (находясь в корне проекта).  
```shell
cargo run --release -- [аргументы]
```

Или перенести куда вам удобно скомпилированный бинарный файл, и запускать его напрямую из терминала.  
```shell
~/<path_to_binary_file> [аргументы]
```

### Примеры запуска в CLI

Все параметры имеют значения по умолчанию, поэтому приложение можно запускать без аргументов — в этом случае будут использованы стандартные настройки.
- **Базовый режим**. Трафик пропускается без модификаций, используется системный DNS-resolver.  
  Минималистичный запуск (используются дефолтные параметры):  
    > ./omoikane
  
  Полная запись (явное указание всех параметров):  
    > ./omoikane -a "127.0.0.1" -p "8080" --dns-mode "system" --log-level "info" --https-split-mode "none" --https-fake-ttl "none"

- **Умеренный режим**. Включает фрагментацию пакетов, Google DoT (IPv4) для обхода простых ограничений.
  > ./omoikane -a "127.0.0.1" -p "8080" --dns-mode "dot" --dns-qtype "ipv4" --https-split-mode "fragment"

- **Максимальный режим**. Использование DNS-over-HTTPS, фрагментации TLS-пакетов и автоматической подстройки TTL.
  > ./omoikane -a "127.0.0.1" -p "8080" --dns-mode "doh" --dns-provider "cloudflare" --dns-qtype "all" --https-split-mode "fragment" --https-fake-ttl "auto"
